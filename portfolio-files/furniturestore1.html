Furniture Store Website

Back-end: Java Server Page, Javascript Front-end: HTML and CSS Database: MySQL

Project Overview

This is my bachelor thesis project. The project is to implement furniture store website that provides commodity recommendation system and transaction system. This website is expected to be a platform for furniture customers and sellers to meet each other and do transaction online.

Tools and Softwares

  • Programming Language: JSP, Javascript, HTML, CSS, MySQL
  • Software: Eclipse, Apache Tomcat, PHPMyAdmin
  • Driver: MySQL-Java connector, commons-io, commons-fileupload

Task Specification

1. Website Information

This website contains:

  • Homepage

    • Homepage is the index of this website. This page contains categorized furniture options and subscription form.

  • Store introduction

    • This page introduce the store's background, contractor, project ideas, and contractor's inquiries.

  • Contact page

    • This page displays staff's contact if users need something to ask about the website or store. Then there is suggestion form on the right side for customer to give suggestion.

  • Customer service page

    • This page is basically a guidance for using the website and store's service. All transaction procedures are in this page.

2. User Management

The user of this website is categorized into 4 types:

  • Administrator

    • Administrator is the master of this website, which makes this user has the highest authority. Administrator can manage user database, product database, broadcast email to subscribers and see all the suggestions sent by customers.

  • Visitor

    • Visitor is categorized as users who doesn't have an account. They can open all the mentioned furniture informations but they have to create an account to access more features such as shopping cart and transactions.

  • Customer

    • Customers are those users who has account and choose 'customer' account type. This type of user can access shopping cart and put some products they would like to buy

  • Seller

    • Seller are those users who created an account as 'seller'. This type of user got different pages than any other user type. Seller can check customer's order requests and delivery progress, manage their own catalogue, and reply customer's comments

3. Commodity Information

This website displays each product with its information such as its material, size, color, price, producer, etc.

4. Commodity Recommendation System

This is the goal of this project. To create an intelligent recommendation system. This system is expected to recommend products to the user based on their preference and search results.

The recommendation system is simple. The website get the search results from users and store it to database. Then 10 most recent search result of one user will be used as query parameter and the result will be displayed as recommendation to the user, like the shown picture above.

5. Transaction System

The website should be the platform where sellers and customers can complete transaction process and comment to each other.

When user request an order, they will be told to adjust their address, recheck their ordered products, and choose the payment method. Then when the transaction is finished, the invoice will be generated through the page and user can print it to pdf format. Finally, this invoice is stored in purchase history page where user can also check the progress of their order request.

Other features

  • Login System

    • Login/Registration system is a must for an e-commerce website. For this website login system, there is a basic data validation for the registration and login, and session creation for later use.

  • Shopping Cart

    • Shopping cart allows customer to put furniture products that they want to buy. From this page user can see and remove products from shopping cart, then proceed to place an order.

  • Search Feature

    • User can search the product based on the commodity information, such as the product's name, material, or even producer.

  • Comment Feature

    • Customer and seller can give comment to a product. Through this feature customer can ask some questions about the product or giving a review, while seller can give feedback through it.
