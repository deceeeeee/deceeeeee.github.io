Furniture Store Website (Remake)

Back-end: PHP Front-end: HTML, CSS, JavaScript, Ajax/jQuery Database: MySQL

Project Overview

This is an updated version of my thesis project. This website is remade with PHP as the back-end. The implementation of jQuery improves the UI and UX of this website.

Tools and Softwares

  • Programming Language: PHP, Javascript, HTML, CSS, MySQL
  • Software: Visual Studio Code, XAMPP
  • Libraries: Font-awesome w3schools, Ajax/jQuery

Updated features

1. Index page

The website layout has changed a little bit since the last project. In this project, the files created for the website has decreased a lot because the implementation of javascript. The index page contains all the furniture informations. Lets see what are the difference!

  • Homepage

    • Homepage is the index of this website. This page now contains furniture category options and recommended product display for visitor and customer users.

  • Store introduction

    • This page introduce the store's background, contractor, and the project ideas. The inquiries has been moved to another side of the page.

  • Customer service page

    • This page does not change much except for the color.

  • Contact

    • The difference between the last project and this for this index page is the footer. This website index now has a footer which contains the inquiries of the store, suggestion form, and subscription form.

2. Login/Register Page

Login and register page have a huge changes too. This is once again because of more implementation of javascript in this website.

  • Login Page

    • Users don't have to access new page to log in anymore. They can simply hover the 'account' tab and another options will show up, which is login and register. If the user click login, the modal box will show up and user can log in.

  • Register Page

    • If the user wants to register an account for this website, they can click the register and they will be redirected to register page. This time, users can choose the account type and the form will be displayed according to which user type that they choose.

3. Catalog Display

The display of the product is slightly different from before. Now users can click the category on the side navigation bar to choose which category they are looking for. Search feature is also available in this page so that users can specifically search for commodities they are looking for.

4. Product Display

This page is also improved and side navigation bar has been added for users to move around in the website easily. The rest is just the same as before.

5. Shopping Cart and Purchase History

Shopping cart UI is now user-friendly. User can now adjust the quantity through the shopping cart page by clicking the button or clicking the quantity of the products that they put in the shopping cart.

Besides shopping cart, purchase history UI has changed also. The list of purchase history looks more interesting than before. Instead of displaying by words, the progress of the order is now displayed by color. White for pending, yellow for delivered, green for received and red for cancelled.

6. Seller Page

Seller page doesn't change much since the last project. The purpose of seller account page is still the same. But now, sellers can do a live search for their own catalog.

7. Administrator Dashboard

Administrator page is now a better, user-friendly page. The layout, table, and everything, has been changed to give a comfortable use for the administrator of this website. Admin can now check account's info or product's info without accessing another page.

The create and update form UI for product management has been updated too.
